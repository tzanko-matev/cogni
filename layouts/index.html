Slop testing framework

Cogni is a slop-testing framework.

Slop means badly-written AI- or human-generated code (context: Merriam-Webster). Slop tests are questions about your code; Cogni runs agents repeatedly to measure effort as the codebase evolves and make maintainability comprehensible across your org.

Problem

Slop creeps in. It is hard to measure.

Teams lack an objective way to quantify how understandable a codebase stays as slop accumulates.

Solution

Turn slop tests into a benchmark.

Define questions, run agents repeatedly, and track effort trends across commits with local reports.

  • Repo-specific slop tests in .cogni/config.yml
  • Objective checks with required citations
  • Trend charts for effort, tokens, and time

How it works

From slop tests to trends in three steps.

Keep the workflow simple and repeatable, from config to report.

01

Define

Write slop tests as focused questions tied to your core features.

02

Run

Agents answer repeatedly with citations while Cogni tracks effort, tokens, time, and tool usage.

03

Report

Compare commits and export a report that shows how effort shifts as the codebase evolves.

Daily usage

Targeted slop tests for individual devs.

Run a subset of questions in minutes and trust the citations.

CLI flow
cogni init
cogni run auth-login@agent-fast repo-health
cogni compare --base main
cogni report --range main..HEAD

Run fast checks before a review or release.

Question suite
# .cogni/config.yml
tasks:
  - id: auth-login
    type: qa
    prompt: "How does login token refresh work?"
    eval:
      validate_citations: true
  - id: repo-health
    type: qa
    prompt: "Where are global error boundaries defined?"
    eval:
      validate_citations: true

Keep questions tied to product features you own.
