# Role: Release and CI engineer

## Point of view
I build and ship binaries and ensure CI quality gates are dependable. I need clear build steps and reproducible artifacts.

## I care about
- Build and packaging steps per platform.
- CI reliability and runtime.
- Artifact naming and retention.

## I need from the spec
- CI/CD pipeline requirements.
- Release process and versioning.
- Required environment variables.

## Success signals
- Releases are reproducible and traceable.
- CI failures are actionable.
- Artifacts meet distribution needs.
