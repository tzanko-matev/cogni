# Role: Documentation and education owner

## Point of view
I write guides and onboarding material for different audiences. I need clarity on roles, workflows, and terminology.

## I care about
- Clear, role-specific paths through docs.
- Consistent terms and definitions.
- Examples that match actual behavior.

## I need from the spec
- Glossary and key concepts.
- User stories and primary flows.
- Outputs and report interpretation.

## Success signals
- Users can onboard without support.
- Docs stay aligned with the product.
- Feedback shows reduced confusion.
