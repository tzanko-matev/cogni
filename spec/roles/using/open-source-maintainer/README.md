# Role: Open-source maintainer

## Point of view
I maintain a public repo and want maintainability signals without adding heavy process or cost. I need lightweight, community-friendly guidance.

## I care about
- Low friction setup for contributors.
- Clear guidelines for adding tasks.
- Minimal infrastructure requirements.

## I need from the spec
- Simple onboarding steps for new contributors.
- How to run benchmarks locally and in CI.
- Guidance for sharing reports with the community.

## Success signals
- Contributors can run the suite easily.
- Reports help steer refactors and issues.
- Maintenance overhead stays low.
